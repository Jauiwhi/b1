# 问题三：评价指标设计与优先级划分

> **核心任务**：基于问题二的分类结果，设计评价指标、划分优先级、判断人工复核必要性，并在资源约束场景下进行分析

## 任务目标

1. **优先级划分**：从紧急程度、错分风险、复核必要性三个维度对文件进行高、中、低等级划分
2. **权重设计**：自主构建权重参数，重点考虑三类文件：
   - 主题归属不明确
   - 时效要求较高
   - 涉及资金分配
3. **复核决策**：综合考虑准确性、时效性、经济成本，判断是否需要人工复核
4. **优先顺序**：列出人工复核的优先顺序和具体原因
5. **资源约束**：在三种资源约束场景下比较处理结果差异

## 核心挑战

| 挑战类型 | 具体表现 | 解决思路 |
|----------|----------|----------|
| **多维度评估** | 三个维度难以直接比较 | 权重设计 |
| **B题要求** | 必须重点考虑三类文件 | 关键词检测 |
| **人工决策** | 哪些需要人工复核？ | 阈值判断 |
| **资源有限** | 不同资源下如何处理 | 场景分析 |

## 解决方案详解

### 1. 三维度优先级计算

我们设计了三维度优先级评分体系：

```python
class PriorityCalculator:
    def calculate_priority(self, file_info):
        # 维度一：紧急程度 (权重0.4)
        urgency_score = self._calculate_urgency(content)
        
        # 维度二：错分风险 (权重0.3)
        risk_score = self._calculate_misclassification_risk(category)
        
        # 维度三：复核必要性 (权重0.3)
        review_score = self._calculate_review_necessity(content)
        
        # 综合得分
        overall = (
            0.4 * urgency_score +
            0.3 * risk_score +
            0.3 * review_score
        )
        
        # 确定优先级等级
        if overall >= 0.6: level = '高'
        elif overall >= 0.35: level = '中'
        else: level = '低'
        
        return {
            'urgency_score': urgency_score,
            'risk_score': risk_score,
            'review_score': review_score,
            'overall_score': overall,
            'priority_level': level
        }
```

**为什么权重是0.4 + 0.3 + 0.3？**

| 权重设计 | 理由 |
|----------|------|
| **紧急程度 0.4** | 时效性是文件处理的第一要素，过期文件可能失去价值 |
| **错分风险 0.3** | 错分可能导致重要文件被忽略，需要复核把关 |
| **复核必要性 0.3** | 涉及资金等敏感内容的文件需要人工确认 |

### 2. 紧急程度计算

```python
def _calculate_urgency(self, content):
    score = 0.0
    
    # 紧急关键词检测
    urgency_count = sum(1 for kw in urgency_keywords if kw in content)
    score += min(urgency_count * 0.2, 0.6)  # 最多0.6分
    
    # 时间敏感检测（有明确日期）
    if self._has_time_sensitive_patterns(content):
        score += 0.4
    
    # 紧急关键词存在
    if any(kw in content for kw in urgency_keywords):
        score += 0.3
    
    return min(score, 1.0)  # 封顶1.0

urgency_keywords = [
    '紧急', '急', '限时', '截止', '尽快', '速办', '重要', '优先级',
    '加急', '特急', '立即', '马上', '即刻', '务必', '必须',
    'deadline', 'urgent', 'emergency'
]
```

**为什么这么做**：
- 关键词累计加分，鼓励同时包含多个紧急信号的文件
- 时间敏感模式检测，针对有明确截止日期的文件
- 封顶机制防止极端情况

### 3. 错分风险计算

```python
def _calculate_misclassification_risk(self, category, analysis):
    score = 0.0
    
    # 归属不明确
    if analysis.get('is_ambiguous'):
        score += 0.5
    
    # 多类别特征
    if analysis.get('is_multi_category'):
        score += 0.4
    
    # 本身就是"其他"或"未分类"
    if '其他' in category or '未分类' in category:
        score += 0.5
    
    return min(score, 1.0)

def _check_ambiguity(self, content, category):
    # 内容过短
    if len(content) < 50:
        return True
    
    # 类别名称模糊
    ambiguous_indicators = ['其他', '综合', '通用', '杂项', '未分类']
    if any(ind in category for ind in ambiguous_indicators):
        return True
    
    return False
```

**为什么这么做**：
- 归属不明确的文件本身就可能是错误分类
- 多类别特征说明难以简单归类
- "其他"类是典型的万金油，应该重点关注

### 4. 复核必要性计算

```python
def _calculate_review_necessity(self, content, analysis):
    score = 0.0
    
    # 涉及资金
    if analysis.get('contains_funding'):
        score += 0.5
    
    # 风险关键词累计
    risk_count = sum(1 for kw in risk_keywords if kw in content)
    score += min(risk_count * 0.15, 0.5)  # 最多0.5分
    
    # 有风险指标
    if analysis.get('risk_indicators'):
        score += 0.3
    
    return min(score, 1.0)

funding_keywords = [
    '资金', '预算', '拨款', '经费', '补贴', '补助', '投资', '融资',
    '贷款', '借款', '担保', '抵押', '资产', '成本', '费用', '支出'
]

risk_keywords = [
    '金额', '合同', '协议', '审批', '签字', '盖章', '法律', '资金',
    '审计', '合规', '风险', '责任', '赔偿', '违约', '诉讼', '处罚'
]
```

**为什么这么做**：
- 资金相关是复核的重点对象（符合B题要求）
- 合同、协议等具有法律效力，需要确认
- 风险关键词累计加分，命中越多越需要关注

### 5. 人工复核决策

```python
class ReviewDecisionMaker:
    def should_review(self, priority_result, category):
        # 条件一：高优先级文件必须复核
        if priority_result['priority_level'] == '高':
            return True
        
        # 条件二：综合得分达到阈值
        if priority_result['overall_score'] >= 0.5:
            return True
        
        # 条件三：无法归类的文件必须复核
        if '未分类' in category or 'ERROR' in category:
            return True
        
        return False
```

**为什么这么做**：
- 高优先级文件定义明确，直接进入复核队列
- 综合得分阈值提供了量化标准
- 无法归类的文件本身分类就不可靠

## 创新亮点

### 亮点一：符合B题要求的权重设计

**B题明确要求**：
> "对于'主题归属不明确'、'时效要求较高'、'涉及资金分配'的文件，应在等级划分和权重设置中重点考虑"

**我们的实现**：

| B题要求 | 实现方式 | 代码位置 |
|---------|----------|----------|
| 时效要求较高 | 紧急程度维度 (权重0.4) | `urgency_score >= 0.5` |
| 主题归属不明确 | 错分风险维度 (权重0.3) | `risk_score >= 0.5` |
| 涉及资金分配 | 复核必要性维度 (权重0.3) | `review_score >= 0.5` |

**效果**：三个维度明确对应B题的三类重点文件

### 亮点二：纯规则实现，高效可解释

