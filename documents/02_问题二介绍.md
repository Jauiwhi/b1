# 问题二：新数据分类与分类效果评价

> **核心任务**：将数据集2和数据集3的新数据纳入问题一建立的分类体系，设计评价指标评估分类效果

## 任务目标

1. **数据集2处理**：对1001个半结构化记录数据进行分类
2. **数据集3处理**：将3518行xlsx数据转换为txt文件并分类
3. **分类体系复用**：基于问题一建立的分类体系对新数据分类
4. **效果评价**：设计评价指标对分类效果进行综合量化评价
5. **边界处理**：识别和处理多类别特征或无法归类的数据

## 核心挑战

| 挑战类型 | 具体表现 | 解决思路 |
|----------|----------|----------|
| **分类一致性** | 新数据必须归入已知类别，不能自创类别 | 提示词约束 |
| **半结构化数据** | xlsx表格需要转换为可处理格式 | 格式转换处理 |
| **评价指标设计** | 无监督分类场景下如何评估 | 聚类质量指标 |
| **边界数据识别** | 同时具有多个类别特征的文件 | 关键词检测 |

## 解决方案详解

### 1. 分类体系复用策略

问题二直接复用问题一建立的分类体系，这是保证分类一致性的关键：

```python
CLASSIFICATION_PROMPT = """
```

### 3. 分类效果评价指标设计

这是本项目的**核心创新点**之一。传统的有监督分类使用Precision/Recall/F1，但这些指标需要**真实标签（Ground Truth）**进行对比。对于无监督分类，没有真实标签，因此需要使用聚类质量评估指标。

#### 为什么不能用传统的Precision/Recall/F1？

```python
# 传统的P/R/F1计算需要：
TP = 预测正确且真实为正
FP = 预测为正但真实为负
FN = 预测为负但真实为正

Precision = TP / (TP + FP)  # 需要知道真实标签
Recall = TP / (TP + FN)     # 需要知道真实标签
F1 = 2 * P * R / (P + R)   # 需要知道真实标签

# 但是：无监督分类没有真实标签！
# 我们不知道文件的"正确答案"是什么类别
```

#### 我们使用的评价指标

| 指标 | 计算方法 | 含义 |
|------|----------|------|
| **准确率 (Accuracy)** | 有效分类文件数 / 总文件数 | 分类覆盖率 |
| **纯度 (Purity)** | 类别内高频词占比 | 类内一致性 |
| **轮廓系数** | (类间分离度 - 类内紧密度) / max(两者) | 聚类质量 |
| **可解释性** | 有效分类文件数 / 总文件数 | 分类可理解性 |
| **迁移适用性** | 分类多样性 * 0.5 + 有效分类率 * 0.5 | 泛化能力 |

### 4. 纯度(Purity)计算详解

```python
def _calculate_category_purity(self, predictions, target_category):
    # 1. 获取该类别的所有文件
    category_files = [p for p in predictions if p.category == target_category]
    
    # 2. 统计类别内所有词的频率
    all_words = []
    for f in category_files:
        words = f.content.split()
        all_words.extend([w for w in words if len(w) >= 2])
    
    # 3. 计算词频
    word_freq = count_word_frequency(all_words)
    
    # 4. 高频词（出现次数>=30%的文件数）越多，纯度越高
    high_freq_words = sum(1 for w, c in word_freq.items() 
                          if c >= len(category_files) * 0.3)
    purity = high_freq_words / len(word_freq)
    
    return purity
```

**为什么这么做**：
- 如果同一类别的文件内容相似，它们会有共同的高频词
- 高频词占比越高，说明类内一致性越好
- 纯度范围[0,1]，越接近1越好

### 5. 轮廓系数(Silhouette Score)计算详解

轮廓系数是国际标准的聚类质量评估指标：

```python
def _calculate_silhouette_score(self, predictions):
    for pred in predictions:
        # a_i: 样本到同类其他样本的平均距离（凝聚度）
        a_i = avg_distance_to_same_cluster(pred)
        
        # b_i: 样本到最近其他簇的平均距离（分离度）
        b_i = min(avg_distance_to_cluster(cat) for cat != pred.category)
        
        # s_i: 轮廓系数
        s_i = (b_i - a_i) / max(a_i, b_i)
    
    return avg(s_i)  # 所有样本的平均值
```

**轮廓系数的含义**：

| 值范围 | 含义 | 说明 |
|--------|------|------|
| 1 ~ 0.7 | 优秀 | 类内紧密，类间分离明显 |
| 0.7 ~ 0.5 | 良好 | 聚类结构合理 |
| 0.5 ~ 0.25 | 一般 | 聚类结构较弱 |
| 0.25 ~ 0 | 较差 | 聚类边界不明显 |
| < 0 | 很差 | 样本可能被分到错误的簇 |

**为什么这么做**：
- 轮廓系数是学术界公认的聚类质量评估标准
- 同时考虑类内紧密度和类间分离度
- 无需真实标签，适用于无监督场景

### 6. 多类别特征检测

```python
def _detect_multi_category(content):
    """检测同时具有多个类别特征的文件"""
    indicators = {
        '人事': ['人事', '员工', '招聘', '培训', '考核', '薪资'],
        '财务': ['财务', '资金', '预算', '报销', '审计', '账目'],
        '技术': ['技术', '研发', '开发', '系统', '软件', '硬件'],
        '行政': ['行政', '办公', '后勤', '物业', '设备'],
        '法务': ['法律', '合同', '协议', '诉讼', '合规']
    }
    
    matched_categories = []
    for cat, keywords in indicators.items():
        if any(kw in content for kw in keywords):
            matched_categories.append(cat)
    
    # 匹配2个及以上类别，说明具有多类别特征
    return len(matched_categories) >= 2
```

**为什么这么做**：
- 某些文件确实涉及多个业务领域
- 检测出这些文件后可以提醒人工复核
- 避免简单分类导致的信息损失

## 创新亮点

### 亮点一：分类体系复用设计

**问题**：问题二的新数据必须与问题一的分类标准保持一致

**解决方案**：
- 问题一的结果JSON中保存了完整的分类体系和合并映射
- 问题二直接加载并使用这些类别
- 提示词中明确约束"只能使用已知类别，禁止自创类别"

**效果**：
- 避免了分类标准漂移
- 保证了新旧数据的分类一致性
- 便于后续追溯和验证

### 亮点二：无监督分类评价指标

**问题**：无监督分类没有真实标签，如何评估分类效果？

**传统方法的局限**：
- Precision/Recall/F1 需要真实标签
- 直接套用会导致指标无意义

**我们的创新**：
| 问题 | 解决方案 | 理论依据 |
|------|----------|----------|
| 如何评估"分类准不准"？ | 纯度：类内高频词占比 | 类内相似性原理 |
| 如何评估"聚类质量"？ | 轮廓系数：类内vs类间距离 | 国际聚类评估标准 |
| 如何评估"可理解性"？ | 可解释性：有效分类比例 | 信息论 |

### 亮点三：半结构化数据友好处理

**问题**：xlsx表格如何被LLM处理？

**解决方案**：
1. 按行拆分，每行数据转为独立txt
2. 格式化为"字段名: 值"形式
3. 保留所有字段的结构信息

**效果**：
- LLM可以像处理普通文本一样处理
- 便于后续分析和追溯
- 兼容问题一的处理流程

### 亮点四：多类别特征自动检测

**问题**：某些文件涉及多个业务领域，如何识别？

**解决方案**：
- 关键词匹配检测多类别特征
- 标记后提醒人工复核
- 避免简单分类导致的信息损失

**效果**：
- 提高了分类的精细度
- 为人工复核提供了重点关注对象
- 增强了系统的鲁棒性
