# 问题一：多源异构文件特征挖掘与分类

> **核心任务**：对3396个历史真实文件进行特征挖掘，建立分类体系并完成自动分类

## 任务目标

1. **特征挖掘**：从文件内容中提取能"反映文件内容及属性"的关键信息
2. **自动分类**：基于提取结果对文件进行分类
3. **文件整理**：将文件放入对应"主题"的文件夹中

## 核心挑战

| 挑战类型 | 具体表现 | 解决思路 |
|----------|----------|----------|
| **多格式异构** | txt、pdf、docx、jpg、png、xlsx六种格式 | 统一内容提取接口 |
| **内容多样** | 表达方式、文体结构、业务属性各异 | LLM自然语言理解 |
| **无预定义分类** | 需要AI自动建立分类体系 | 提示词工程引导 |
| **标题干扰** | "标题+表格"结构，表格干扰判断 | 掐头去尾+标题优先 |

## 解决方案详解

### 1. 统一内容提取

不同文件格式需要不同的处理方式，我们设计了统一的处理接口：

```python
class BatchClassifier:
    def process_file(self, path):
        if ext == '.txt':   return extract_txt(path)
        if ext == '.pdf':   return extract_pdf(path)
        if ext == '.docx':  return extract_docx(path)
        if ext in images:   return extract_image(path)  # OCR
        if ext == '.xlsx':  return extract_xlsx(path)
```

**为什么这么做**：
- 每种格式的内部结构完全不同，需要专门的解析器
- 统一接口使得上层调用无需关心文件格式差异
- 便于扩展新的文件类型

### 2. 智能提示词设计

分类提示词是分类效果的关键，我们精心设计了以下原则：

```python
CATEGORY_PROMPT = """
【核心分类原则】
1. 标题优先级最高：很多文件由"顶部标题 + 大段正文/大片表格"组成
   → 请务必优先识别并依据文件最上方的"标题"来界定业务属性
2. 内容优先级高：无论文件原来是Word、Excel还是图片
   → 请一律根据其表达的"实际业务内容"分类
   → 绝对禁止输出"图片素材"、"扫描件"等无意义类别
3. 动态命名：类别名称由你根据内容精准提炼，长度限制在 2-8 个字
4. 特殊兜底：
   - 纯 URL 链接列表 → "链接索引"
   - 内容极度模糊 → "未分类"
"""
```

**为什么这么做**：

| 设计要点 | 原因 | 效果 |
|----------|------|------|
| 标题优先级 | 很多文件的标题就是业务属性的直接表达 | 分类更准确 |
| 禁止图片类别 | AI容易输出无意义的"图片"类别 | 避免无效分类 |
| 2-8字限制 | 太长难以管理，太短无法表达 | 平衡可读性 |
| 兜底机制 | 边界情况需要明确处理方式 | 提高鲁棒性 |

### 3. 上下文智能截断

当文件内容过长时，我们采用**掐头去尾保留法**：

```python
def _truncate_for_context(self, text, max_chars=800):
    head_chars = int(max_chars * 0.6)  # 保留首部60%
    tail_chars = int(max_chars * 0.3)   # 保留尾部30%
    
    head = text[:head_chars]           # 标题通常在开头
    tail = text[-tail_chars:]           # 结论/摘要通常在结尾
    
    return f"{head}\n...\n{tail}"
```

**为什么这么做**：
- **首部60%**：通常包含标题、开头段落，是分类的关键依据
- **尾部30%**：通常包含总结、结论，是对内容的收尾
- **中间丢弃**：表格数据、重复内容通常在中间，不影响分类

LLM初次分类可能产生语义相近的类别，需要合并：

## 创新亮点

### 亮点一：标题优先级识别策略

**问题**：很多文件是"标题+表格"结构，表格数据会干扰分类判断

**解决方案**：提示词中明确强调"标题优先级最高"，并配合掐头去尾截断法

**效果**：确保分类依据的是业务主题而非表格数据

### 亮点二：禁止万金油类别约束

**问题**：LLM容易输出"综合"、"通用"等无意义类别

**解决方案**：在提示词中明确禁用词库，要求必须是具体的业务实体

**效果**：分类结果更具业务价值和可解释性

### 亮点三：双重保底合并机制

**问题**：LLM可能失败或输出不稳定

**解决方案**：
1. 主方案：LLM深度聚类合并
2. 保底方案：本地启发式合并（关键词匹配）

**效果**：无论何种情况都能得到合理的合并结果

### 亮点四：上下文打包优化

**问题**：直接截断可能丢失关键信息

**解决方案**：
- 掐头去尾保留首尾关键信息
- 智能打包充分利用上下文窗口
- 批量处理减少API调用

**效果**：分类速度提升数倍，成本大幅降低
