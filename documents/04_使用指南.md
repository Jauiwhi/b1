## 安装步骤

### 如果使用 start.bat（推荐）
**你只需要配置API Key，其他一切自动！**

1. 用记事本打开 [config.yaml](file:///c:/Users/Administrator/Desktop/b1/config.yaml)
2. 填入你的API Key（见下方配置说明）
3. 双击 [start.bat](file:///c:/Users/Administrator/Desktop/b1/start.bat) 即可

### 手动安装（非Windows用户）

在项目根目录下执行：

```bash
pip install -r requirements.txt
```

如果需要OCR功能，还需要安装：

```bash
pip install pytesseract
```

**注意**：OCR功能需要安装Tesseract OCR引擎：
- **Windows**：下载并安装 [Tesseract OCR](https://github.com/UB-Mannheim/tesseract/wiki)
- **macOS**：`brew install tesseract`
- **Linux**：`sudo apt-get install tesseract-ocr`

### 第二步：安装LLM模型（可选）

#### 方案A：使用云端模型（推荐新手）

直接配置API Key，无需安装本地模型。

支持的云端模型：
| 模型 | 配置项 | API获取地址 |
|------|--------|-------------|
| DeepSeek | `deepseek.api_key` | https://platform.deepseek.com |
| Qwen | `qwen.api_key` | https://dashscope.console.aliyun.com |
| ChatGPT | `chatgpt.api_key` | https://platform.openai.com |
| Gemini | `gemini.api_key` | https://aistudio.google.com |

#### 方案B：使用本地模型

##### Ollama（推荐）

1. 下载安装 [Ollama](https://ollama.com/)
2. 启动Ollama服务：
   ```bash
   ollama serve
   ```
3. 下载模型：
   ```bash
   ollama pull llama3.2
   ```

##### LM Studio

1. 下载安装 [LM Studio](https://lmstudio.ai/)
2. 启动LM Studio并加载模型
3. 确认API服务地址（默认：`http://localhost:1234`）

## 配置说明

打开 [config.yaml](file:///c:/Users/Administrator/Desktop/b1/config.yaml) 文件进行配置：

### 配置示例1：使用DeepSeek云端模型（最推荐）

```yaml
model_provider: deepseek

cloud:
  deepseek:
    api_key: "your-deepseek-api-key-here"  # 替换为你的API Key
    model: deepseek-chat
    base_url: https://api.deepseek.com/v1
```

### 配置示例2：使用本地Ollama

```yaml
model_provider: ollama

local:
  host: localhost
  port: 11434
  model_name: llama3.2
  timeout: 300
```

### 配置示例3：使用本地LM Studio

```yaml
model_provider: lm_studio

local:
  host: localhost
  port: 1234
  model_name: local-model
  timeout: 300
```

## 命令行参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--problem 1/2/3/all` | 指定运行的问题 | `--problem all` |
| `--config <path>` | 指定配置文件 | `--config my_config.yaml` |
| `--dataset1 <path>` | 指定数据集1路径 | `--dataset1 ./data/dataset1` |
| `--dataset2 <path>` | 指定数据集2路径 | `--dataset2 ./data/dataset2` |
| `--dataset3 <path>` | 指定数据集3路径 | `--dataset3 ./data/dataset3.xlsx` |
| `--dataset4 <path>` | 指定数据集4路径 | `--dataset4 ./data/dataset4.xlsx` |
| `--output <path>` | 指定输出目录 | `--output ./my_output` |
| `--image-mode` | 图片识别模式 | `--image-mode ocr` |
| `--interactive` | 交互式模式 | `--interactive` |

### 图片识别模式选项

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `vision` | AI视觉识别 | 云端模型（GPT-4V、Gemini） |
| `ocr` | OCR文字识别后分类 | 本地模型、通用场景 |
| `skip` | 跳过图片 | 仅处理文本文件 |

## 常见问题

### Q1：提示 "Model connection failed"

**原因**：模型连接失败

**解决方法**：
1. 检查 [config.yaml](file:///c:/Users/Administrator/Desktop/b1/config.yaml) 中的API Key是否正确
2. 检查网络连接
3. 如果使用本地模型，确认服务是否启动

### Q2：提示 "Dataset path not found"

**原因**：数据集路径不存在

**解决方法**：
1. 确保数据集文件夹/文件放在 `input/` 目录下
2. 或使用 `--dataset1` 等参数指定正确路径

### Q3：OCR识别结果不准确

**解决方法**：
1. 安装中文语言包：`https://github.com/tesseract-ocr/tessdata`
2. 将语言包文件放入Tesseract安装目录的`tessdata`文件夹
3. 重启程序

### Q4：处理速度很慢

**优化建议**：
1. 使用云端模型（通常更快）
2. 减少 [config.yaml](file:///c:/Users/Administrator/Desktop/b1/config.yaml) 中的 `batch_size`
3. 使用GPU加速的本地模型

### Q5：内存不足

**解决方法**：
1. 减小 [config.yaml](file:///c:/Users/Administrator/Desktop/b1/config.yaml) 中的 `batch_size`
2. 减小 `max_context_tokens`（默认32000）
3. 使用更小的模型
